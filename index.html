vue路由传参query与params

2019-11-01

params：动态路径参数，以冒号开头（如path: ‘user/:id/admin’）-> (url：user/123/admin)，并且该参数值会被设置到 $route.params
query：查询参数，是拼在url后边的（url：user?id=123）

注：（对于params动态路径参数）
1、如果路由path配置动态参数（:id），可以使用能提供路由的 name，并配置params或手写完整的带有参数的 path。
2、如果路由path配置动态参数（:id），使用手写完整的带有参数的 path的话，又配置另外了params参数，该params会被忽略，$route.params也只能拿到path配置的参数。
3、如果路由path配置动态参数（:id），使用提供路由的 name时又不配置了params参数的话，生成的url就会出错。

5、如果路由path配置动态参数（:id），使用提供路由的 name，又配置了params参数的话，页面跳转后可以拿到params所有参数，但是刷新页面，params除了path配置的动态参数不会丢失，其他都会参数会丢失。
6、如果路由path没有配置动态参数（:id），使用提供路由的 name，又配置了params参数的话，页面跳转后可以拿到params，但是刷新页面params参数会丢失。

无论路由path是否有配置动态参数，使用path（手写完整的 path）的话，如另外配置params参数，该params都会被忽略。

1、路由path配置了动态参数，params传参

2、路由path无动态参数，params传参

3、query（无论路由path是否有配置动态参数，query参数都是拼在url后边的）

总结：使用路由时用 name 和 path 都可以。用 name 的话，如果有设置路径参数，需要额外加上 params；用path的话，path 可以写完整路径（包括参数）, path 的优先级更高，写了 path ，params 这些会被忽略。
