2019-11-06

为什么需要懒加载
在单页应用中，如果没有应用懒加载，运用webpack打包后的文件将会异常的大，造成进入首页时，需要加载的内容过多，延时过长，不利于用户体验，而运用懒加载则可以将页面进行划分，需要的时候加载页面，可以有效的分担首页所承担的加载压力，减少首页加载用时

什么是懒加载
懒加载也叫延迟加载，即在需要的时候进行加载，随用随载。

vue是如何实现路由组件懒加载的

Vue 的异步组件

在大型应用中，我们可能需要将应用分割成小一些的代码块，并且只在需要的时候才从服务器加载一个模块。

Vue 只有在这个组件需要被渲染的时候才会触发该工厂函数，且会把结果缓存起来供未来重渲染。

Webpack代码分割

能够把代码分离到不同的 bundle 中，然后可以按需加载或并行加载这些文件。代码分离可以用于获取更小的 bundle，以及控制资源加载优先级，如果使用合理，会极大影响加载时间。

常用的代码分离方法有三种：

入口起点：使用 entry 配置手动地分离代码。
防止重复：使用 SplitChunksPlugin 去重和分离 chunk。
动态导入：通过模块中的内联函数调用来分离代码。

1、入口起点

这种方式存在一些隐患：

如果入口 chunk 之间包含一些重复的模块，那些重复模块都会被引入到各个 bundle 中。
这种方法不够灵活，并且不能动态地将核心应用程序逻辑中的代码拆分出来。

2、防止重复
在入口起点方法中造成一些模块在两个 bundle 中重复引用。
我们可以通过使用 SplitChunksPlugin 插件来移除重复模块。

SplitChunksPlugin插件可以将公共依赖模块提取到已有的entry chunk中，或者提取到一个新生成的chunk。

使用 optimization.splitChunks 配置选项，现在可以看到已经从 index.bundle.js 和 another.bundle.js 中删除了重复的依赖项。需要注意的是，此插件将 lodash 这个沉重负担从主 bundle 中移除，然后分离到一个单独的 chunk 中。

3、动态导入
动态代码拆分webpack 提供了两个类似的技术：使用es6 import()语法；使用 webpack 的遗留功能，webpack特定的require.ensure
（1）、使用es6 import()语法
import() 特性依赖于内置的 Promise，动态地加载模块。
调用 动态import() 之处，被作为分离的模块起点，意思是，被请求的模块和它引用的所有子模块，会分离到一个单独的 chunk 中。

注意，在注释中我们提供了 webpackChunkName。这样会将拆分出来的 bundle 命名为 lodash.bundle.js，而不是 [id].bundle.js。

（2）、使用webpack特定的require.ensure
require-ensure在需要的时候才下载依赖的模块；
当参数指定的模块都下载下来了（下载下来的模块仅仅是可被使用，并没有被执行），便执行参数指定的回调函数。
require-ensure会创建一个chunk，且可以指定该chunk的名称，如果这个chunk名已经存在了，则将本次依赖的模块合并到已经存在的chunk中，最后这个chunk在webpack构建的时候会单独生成一个文件。

注：require.ensure 内部依赖于 Promises。

vue-router：导航守卫

2019-11-05

vue-router 提供的导航守卫主要用来通过跳转或取消的方式守卫导航。有多种机会植入路由导航过程中：全局的, 单个路由独享的, 或者组件级的。

你可以通过观察 $route 对象来应对这些变化，或使用 beforeRouteUpdate 的组件内守卫。

完整的导航解析流程

1、导航被触发。
2、在失活的组件里调用离开守卫。
3、调用全局的 beforeEach 守卫。
4、在重用的组件里调用 beforeRouteUpdate 守卫 (2.2+)。
5、在路由配置里调用 beforeEnter。
6、解析异步路由组件。
7、在被激活的组件里调用 beforeRouteEnter。
8、调用全局的 beforeResolve 守卫 (2.5+)。
9、导航被确认。
10、调用全局的 afterEach 钩子。
11、触发 DOM 更新。
12、用创建好的实例调用 beforeRouteEnter 守卫中传给 next 的回调函数。

vue-router：路由组件传参

2019-11-05

在组件中使用 $route 会使之与其对应路由形成高度耦合，从而使组件只能在某些特定的 URL 上使用，限制了其灵活性。

如：有一个子组件需要用到参数(id)，在下案例中可以通过$route.params拿到；
但是如果该子组件在某个父组件中直接引入使用占位符的形式插入父组件时，该子组件就无法通过$route.params拿到参数（id）了。

使用 props 将组件和路由解耦，取代与 $route 的耦合，该props有三种模式：布尔模式、对象模式、函数模式

布尔模式

props 被设置为 true，route.params 将会被设置为组件属性（props）。

对象模式

props 是一个对象，它会被按原样设置为组件属性。当 props 是静态的时候有用。

函数模式

你可以创建一个函数返回 props。这样你便可以将参数转换成另一种类型，将静态值与基于路由的值结合等等。

注：请尽可能保持 props 函数为无状态的，因为它只会在路由发生变化时起作用。如果你需要状态来定义 props，请使用包装组件，这样 Vue 才可以对状态变化做出反应。

vue-router：params与query传参

2019-11-01

Vue-router 做路径匹配时支持动态片段、全匹配片段以及查询参数（片段指的是 URL 中的一部分）。

params：是包含了动态片段和全匹配片段的动态路径参数，并且该参数值会被设置到 $route.params；params一旦设置在路由，params就是路由的一部分，必须要有；如果没有路由参数，就是一个空对象。
动态片段：以冒号开头（如path: ‘user/:id/admin’）-> (url: user/123/admin)；
全匹配片段: 想匹配任意路径，我们可以使用通配符(*)；如（path: ‘*‘，会匹配所有路径）并通常用于客户端 404 错误，或（path: ‘/user-*‘，会匹配以 /user- 开头的任意路径）。

query： 查询参数，是拼在url后边的（url: user?id=123）

注：（对于params动态路径参数）
1、如果路由path配置动态参数（:id），可以使用能提供路由的 name，并配置params或手写完整的带有参数的 path。
2、如果路由path配置动态参数（:id），使用手写完整的带有参数的 path的话，又配置另外了params参数，该params会被忽略，$route.params也只能拿到path配置的参数。
3、如果路由path配置动态参数（:id），使用提供路由的 name时又不配置了params参数的话，生成的url就会出错。

5、如果路由path配置动态参数（:id），使用提供路由的 name，又配置了params参数的话，页面跳转后可以拿到params所有参数，但是刷新页面，params除了path配置的动态参数不会丢失，其他都会参数会丢失。
6、如果路由path没有配置动态参数（:id），使用提供路由的 name，又配置了params参数的话，url不会显示该params参数，页面跳转后可以拿到params，但是刷新页面params参数会丢失。

url会显示经（:）或（*）配置的params参数，不会显示未经（:）或（*）配置的params参数，并且未经（:）或（*）配置的params参数在页面刷新后都会丢失。

1、params动态片段

2、路由path配置了动态路径参数，params传参

3、路由path无动态路径参数，params传参

4、query（无论路由path是否有配置动态参数，query参数都是拼在url后边的）

总结：使用路由时用 name 和 path 都可以。用 name 的话，如果有设置动态路径参数，需要额外加上 params；用path的话，path 可以写完整路径（包括参数）, path 的优先级更高，写了 path ，params 这些会被忽略。
